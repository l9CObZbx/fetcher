# fetcher-llm-streaming

Implement streaming features for LLM APIs using Fetcher's eventstream package.

## Trigger Conditions

This skill activates when the user:
- Wants SSE/streaming support
- Mentions LLM, OpenAI, streaming, eventStream, or jsonEventStream
- Asks about real-time data or token streaming

## Side-Effect Import Pattern

The eventstream package uses a side-effect import to extend the global `Response.prototype`:

```typescript
import '@ahoo-wang/fetcher-eventstream';
```

This import automatically adds event stream methods to all Response objects with `text/event-stream` content type.

## Response Prototype Extension

After importing the side-effect module, Response objects gain:

| Method | Returns | Description |
|--------|---------|-------------|
| `isEventStream` | `boolean` | True if Content-Type is `text/event-stream` |
| `eventStream()` | `ServerSentEventStream \| undefined` | Converts to SSE stream |
| `requiredEventStream()` | `ServerSentEventStream` | Throws if not available |
| `jsonEventStream<DATA>()` | `JsonServerSentEventStream<DATA> \| undefined` | Converts to typed JSON stream |
| `requiredJsonEventStream<DATA>()` | `JsonServerSentEventStream<DATA>` | Throws if not available |

## SSE Parsing

Server-Sent Events are parsed according to the specification with the following fields:

```typescript
interface ServerSentEvent {
  data: string;    // Event data (required)
  event?: string;  // Event type (default: 'message')
  id?: string;     // Event ID for tracking
  retry?: number;  // Reconnection timeout in ms
}
```

## Termination Detection

Clean stream cleanup using `TerminateDetector`:

```typescript
import { toJsonServerSentEventStream, type TerminateDetector } from '@ahoo-wang/fetcher-eventstream';

// OpenAI-style termination
const terminateOnDone: TerminateDetector = event => event.data === '[DONE]';

const jsonStream = toJsonServerSentEventStream<ChatCompletionChunk>(
  response.requiredEventStream(),
  terminateOnDone,
);
```

Common termination patterns:
- OpenAI: `event.data === '[DONE]'`
- Event-based: `event.event === 'done'`
- Multi-condition: Combine event type and data checks

## OpenAI Client Streaming

```typescript
import { OpenAI } from '@ahoo-wang/fetcher-openai';

const openai = new OpenAI({
  baseURL: 'https://api.openai.com/v1',
  apiKey: process.env.OPENAI_API_KEY!,
});

// Streaming chat completion
const stream = await openai.chat.completions({
  model: 'gpt-3.5-turbo',
  messages: [{ role: 'user', content: 'Hello!' }],
  stream: true,
});

// Process tokens as they arrive
for await (const chunk of stream) {
  const content = chunk.choices[0]?.delta?.content || '';
  if (content) {
    process.stdout.write(content); // Real-time output
  }
}
```

## Real-Time UI Update Patterns

### Token-by-Token Updates

```typescript
let fullResponse = '';

for await (const event of stream) {
  if (event.data) {
    const chunk = event.data;
    const content = chunk.choices[0]?.delta?.content || '';
    fullResponse += content;
    updateUI(content); // Real-time UI update
  }
}
```

### Progress Tracking

```typescript
let chunksReceived = 0;
let totalContent = '';

for await (const chunk of stream) {
  chunksReceived++;
  const content = chunk.choices[0]?.delta?.content || '';

  if (content) {
    totalContent += content;
    if (chunksReceived % 5 === 0) {
      console.log(`Received ${chunksReceived} chunks`);
    }
  }

  if (chunk.choices[0]?.finish_reason) {
    console.log(`Finished: ${chunk.choices[0].finish_reason}`);
    break;
  }
}
```
